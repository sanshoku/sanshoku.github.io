我發現我的電腦語言太多了一直處在混亂邪惡狀態，與其期待他跑出正常的結果不如寫篇教學文章（讀音：誤導廢文）來講解vue的使用方式。
有很多人在接觸前端的時候都止步於html跟css，主要的工作內容就是切版（順帶一提我回到台灣才第一次聽到切版，第一個反應就是這三小，後來才知道是畫盒子），不會接觸到Javascript相關的東西。 對我來說很新奇的是，雖然在網頁開發上我是超級新手，學校的教育也沒很想把我們這些UX designer養成什麼都會的獨角獸，但簡單帶過html跟css以後我們就⋯⋯都在動態網頁。

Vue是我自己最喜歡的，因為他最簡單最可愛。（損友說是電波對到了）
間單的說，Vue是React, Angular, Vue三個主要的前端模板裡最好學也是最輕便的
對於有點基本網頁code經驗的人而言理論上不難，真的，超基本就可以

那Vue是啥東西呢？他到底為啥比html好用咧？

首先既然要寫教學文就要準備教學的內容
要用Vue就要下載Vue，我自己習慣的編譯器是VS studio，免費的，大家都可以去載個。會換VS純粹因為當年（？）學Python的時候因為名字很酷所以很喜歡的Spyder不知道為什麼就是裝不進我的電腦（我不當工程師的主要原因之一就是我常常觸發一些奇蹟般的bug，拿去問在iBM工作的朋友他都摸不著頭緒的那種）
怎麼裝東西網路上應該有不少文章，不過我是覺得不用講這麼複雜，裝個node.js然後在cmd裡（蘋果叫終端機）打(sudo) npm install -g @vue/cli 讓他跑就好了

基本上vue的邏輯就是，你開一個範本以後script的內容會被餵給範本，有些人叫這個「渲染」。這種做法叫前端渲染，好處是你伺服器不需要一直吞吞吐吐也不會改了個模板就到處爆炸，壞處是SEO抓動態網頁比較慢讀網頁也可能稍微久一點。就我自己的立場來說我是覺得「那不然我自己讓他跑成html然後抓下來XD」

很多文章說Vue改起來比較容易，原因之一也有他可以建立components（也不是說別的就不行，反正程式語言都長很像）然後插在不同的網頁裡。components 的作法就是寫好一個.vue檔然後直接引用到要用的動態網頁裡，舉例來說我們有個檔案叫card.vue好了，要把這個card嵌在網頁裡只要喊他就好

是一樣的。你可能會注意到這堆東東裡沒有data，這是因為在引用的時候你需要另一張卡片來存資料屬性，跟你本人到公司面試還是要帶張履歷在身上同樣的道理。為了保存這些data，我們需要建立一張叫做props或emit的卡卡來跟這個卡卡給template讀，props是從外往內，吃data，emit則是比較接近事件

Props一般來說長這樣，先寫變數名稱然後再宣告個屬性以防他爆炸。爆炸的意思是，你明明餵了data但你的程式不知道該怎麼對答案。

除此之外，在component 中的data必須是公式
如果data不是公式而是值，你在同一頁上貼的組件就會share一個變數的格子

當然你也可以在同一個檔案裡宣告這個組件，也是個人練習時比較常見的做法，
這樣就不用引來引去，像我這樣電腦路徑特別容易爆又不是很擅長管理網頁的人也不用擔心搞爆⋯⋯

要把數據綁在裡面的時候就在引用以後 加個:你的資料形狀
以上面那個例子來說就是 <component名城 :posts = “posts”/>
